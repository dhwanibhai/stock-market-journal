Documenting the Journey of Our Stock Market Simulator Project

Project Timeline

  • October 30: Formed teams and finalized the idea for StockSim. Outlined the core functionalities such as real-time trading, portfolio tracking, and watchlist features.
  • November 6: Met to discuss the layout and basic planning. Decided on a modular architecture to facilitate development and integration.
  • November 8: Implemented a secure login system using GitHub integration for authentication. Tested the system for edge cases, such as invalid credentials and multiple logins.
  • November 10: Integrated the Alpha Vantage free API to fetch stock market data. Debugged an issue where API keys were incorrectly configured, delaying implementation.
  • November 12: Began to utilize Alpha Vantage's closing prices for market analysis and stock trading. Improved error handling for API rate limits.
  • November 13-16: Built key features including portfolio management, positions tracking, funds management, the main menu interface, and a watchlist for favorite stocks.
  • November 18: Attempted integration with Angel One API but faced unclear documentation. Pivoted to exploring alternative solutions.
  • November 20: Purchased the Kite Connect API for â‚¹2000 to access live market data. Set up and tested API integration. Encountered initial performance issues due to delayed responses.
  • November 22: Modified all functions to work with live market data from Kite Connect API. Encountered compilation issues when consolidating multiple functions into one main function.
  • November 26: Met with Prajish Sir to explore working with header files for a modular code structure. Learned to manage interdependencies across modules.
  • November 27: Resolved a device-specific issue where the Kite Connect API was not functioning correctly. Completed a fresh setup on the affected device.
  • November 30: Created `.h` files for all functions, consolidated the main function into a 3-liner main, and resolved multiple bugs during the debugging process.
  • December 1: Addressed remaining compilation issues using a Makefile to streamline the build process and enable minimal commands for integration.
  • December 3: Introduced multiple user functionalities and dynamic portfolio views with non-blocking mode to enhance user experience without clearing the terminal.
  • December 5: Implemented a method to map user-entered trading symbols to their corresponding `instrument_token` values required by the API. This involved creating a process to convert the user-friendly input into the specific format needed for accurate data retrieval and transaction execution through the API.
  • December 6: Added trade history and timestamps for transactions. Fixed additional bugs reported during internal testing.
  • December 7: Developed functionality to create a user file automatically when a new user is added, ensuring better user data management.
  • December 8: Created the README file to document the project, including installation steps, dependencies, and a usage guide.
  • December 8: We added support for lot sizes so users can trade in predefined quantities based on market rules. This makes sure that the trades are realistic and match how things work in the actual stock market.
